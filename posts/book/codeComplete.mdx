# 코드 컴플리트

## 1장 : 소프트웨터 구현으로의 초대

소프트웨어 구현이란?
개발 과정의 중심적인 활동으로 주요 활동은 상세 설계, 코드 작성, 디버깅 통합, 개발자 테스트다.
구현 품질은 소프트웨어의 품질에 많은 영향을 준다.

문제정의, 요구사항 개발, 구현계획 수립, 소프트웨어 아키텍처 또는 고급수준의 설계, 상세 설계, 코드 작성 및 디버깅, 단위 테스트, 통합 테스트, 통합, 시스템 테스트, 유지보수

## 2장 소프트웨어 개발자를 돕기 위한 비유

소프트웨어 구현을 건축 공사와 비슷하게 취급하면 신중한 준비 과정이 필요하다는 것을 알려주고, 크고 작은 프로젝트 사이의 차이점을 보여준다.
소프트웨어 개발 실천법을 지적 도구 상자에 있는 도구로 생각하면 모든 개발자가 많은 도구를 갖고 있고 하나의 도구로 모든 일으 처리할 수 없다는 것을 말해준다. 문제마다 그에 맞는 도구를 선택하는 것이 효과적인 개발자가 되는 비결이다.

## 3장 철저하게 준비하게: 선행 조건

구현 전에 선행 조건을 수행하기 위한 필수적인 논의

1. 논리적 설득
2. 비유적 설득
3. 데이터에 근거한 설득

방법

1. 반복적인 방법

- 요구사항을 제대로 이해할 수 없거나 다른 여러가지 이유 때문에 변경될 가능성이 많을 때
- 설계가 복잡하거나 어려울 때
- 개발 팀이 해당 응용 분야에 대해 잘 모를 때
- 프로젝트의 위험 부담이 높을 때
- 장기적인 계획이 중요하지 않을 때
- 요구사항, 설계, 코드 변경 비용이 높지 않을 것 같을 때

2. 순차적인 방법

- 요구사항이 안정적일 때
- 설계가 직관적이며 이해하기 쉬울 때
- 개발 팀이 해당 응용 분야에 익숙할 때
- 프로젝트의 위험 부담이 적을 때
- 장기적인 계획이 중요할 때
- 요구사항, 설계, 코드 변경 비용이 높을 것 같을 때

조건

1. 문제-정의 선행 조건
2. 요구사항 선행 조건 > 명시적인 요구사항
3. 아키텍처 선행 조건

- 아키텍처의 목표는 명확하게 기술되어야 한다.
- 모든 주요 결정사항에 대한 동기를 기술해야 한다.
- 명시적으로 위험 영역을 규명해야 한다. 그것이 왜 위험하고 위험을 최소화 하기 위해서 어떤 단계를 밟아야 하는지 설명해야 한다.

## 4장 구현 시 결정해야 할 핵심 사항

1. 프로그래밍 언어 선택

- 언어의 장단점을 파악한 후 이를 바탕으로 언어를 선택한다.
- 언어에 의한 프로그램이 아니라 언어를 활용한 프로그래밍이 중요하다.

2. 프로그래밍 규약 : 반드시 프로그래밍을 시작하기 전에 규약을 만들어야 리소스 낭비가 적다.
3. 기술 흐름 파악 : 기술적인 흐름에서의 위치에 따라서 어떠한 접근 방법이 더 효과적인지 살펴본다.
4. 구현 실전법 선택
